Madd Assembler

1. Introduction

The assembler translates assembly language - a human-readable representation of machine instructions, into binary machine code, directly executable by the CPU. The machine code (and any data) is placed in an object file in the format appropriate for the target machine, which may be passed to the linker to create executable or libraries. Assembly language is different on each CPU architecture.

1.1. Command Usage

 
[prefix-]as [-o output-file] input-file

The input-file parameter specifies the name of a file from which assembly code will be read; if it is not specified, the assembler by defualt reads from standard input. The output-file parameter names the object file into which assembled data will be written; if not specified, it default to a.out.
